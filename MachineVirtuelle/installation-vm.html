Installation de la Machine Virtuelle

Installation et Configuration

1. Créer la "boîte" (la Machine Virtuelle)

Tout commence dans VirtualBox. J'ai cliqué sur "Nouvelle" pour créer ma machine. Comme on le voit sur ma capture, j'ai choisi le nom "Ubuntu" et j'ai séléctionné l'image ISO ubuntu-24.04.3-desktop-amd64.iso que j'avais téléchargée.

2. Booster les performances

Pour que ma machine ne rame pas, je suis allé dans les réglages système :

La RAM : J'ai mis le curseur à 8 Go. C'est large, mais ça permet de faire tourner le serveur et mes outils de code en même temps.
Le Processeur : J'ai alloué 10 CPU. Avec ça, l'installation d'Ubuntu et les futures mises à jour se font hyper rapidement.

Configuration des utilisateurs

Pour administrer mon système de manière professionnelle, on a mis mis en place une structure multi-utilisateurs :

  • Nous avons créé un groupe nommé BinomeG5 pour organiser les accès de mon équipe.
    • sudo groupadd BinomeG5
  • Nous avons créé deux comptes standards, BinomeG5_1 et BinomeG5_2, pour les tâches de développement courantes.
    • sudo adduser BinomeG5_1 --allow-bad-names
    sudo adduser BinomeG5_2 --allow-bad-names
    sudo usermod -aG BinomeG5 BinomeG5_1
    sudo usermod -aG BinomeG5 BinomeG5_2
  • Nous avons créé le compte admin_Gr103_BinomeG5. Je l'ai ajouté au groupe sudo pour lui donner les droits d'administration.
    • sudo adduser admin_Gr103_BinomeG5 --allow-bad-names
    sudo usermod -aG sudo admin_Gr103_BinomeG5
    sudo usermod -aG BinomeG5 admin_Gr103_BinomeG5
  • Comme nous avons utilisé des majuscules dans mes noms d'utilisateurs, j'ai dû utiliser l'option --allow-bad-names pour passer les restrictions de Linux.

Enfin, j'ai personnalisé mon invite de commande pour qu'elle affiche mon nom d'utilisateur. En modifiant le fichier .bashrc de chaque compte, j'ai rendu mon environnement plus clair : je sais désormais instantanément quel utilisateur est actif dans le terminal.

nano ~/.bashrc
PS1="\u > "
source ~/.bashrc
